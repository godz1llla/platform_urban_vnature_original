## 🚀 Быстрое развертывание на сервере

### Шаг 1: Установка зависимостей

```bash
# Обновите систему
sudo apt update && sudo apt upgrade -y

sudo apt install -y nginx mysql-server php-fpm php-mysql php-mbstring php-xml php-curl
```


### Шаг 3: Создание и импорт базы данных

```bash
# Создайте базу данных
```

### Шаг 4: Настройка подключения к БД

Отредактируйте файл `backend/config/Database.php`:

```php
private $host = "localhost";
private $db_name = "urban_college";
private $username = "root";      // ваш MySQL пользователь
private $password = "ваш_пароль"; // ваш MySQL пароль
```

### Шаг 5: Настройка Nginx

Создайте конфигурационный файл:

```bash
sudo nano /etc/nginx/sites-available/urban_college.conf
```

Вставьте следующую конфигурацию:

```nginx
server {
    listen 80;
    server_name ваш_домен.kz;  # замените на ваш домен или IP

    root /home/hubtech/Documents/1orig_URBAN;  # путь к проекту
    index index.html index.php;

    # Логи
    access_log /var/log/nginx/urban_college_access.log;
    error_log /var/log/nginx/urban_college_error.log;

    # Основная локация
    location / {
        try_files $uri $uri/ =404;
    }

    # PHP обработка
    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.2-fpm.sock;  # проверьте версию PHP
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    # API маршрут
    location /backend/ {
        try_files $uri $uri/ =404;
    }

    # Защита от доступа к скрытым файлам
    location ~ /\. {
        deny all;
    }
}
```

Активируйте конфигурацию:

```bash
# Создайте символическую ссылку
sudo ln -s /etc/nginx/sites-available/urban_college.conf /etc/nginx/sites-enabled/

# Проверьте конфигурацию
sudo nginx -t

# Перезапустите Nginx
sudo systemctl reload nginx
```

## 🔧 Критические исправления

### Общие исправления модальных окон

**Zombie Overlay Killer Pattern:**
```javascript
function createCleanModal() {
    // 1. Убиваем все старые модалки
    const old = document.querySelectorAll('.dynamic-modal-overlay');
    old.forEach(el => el.remove());
    
    // 2. Создаем новую чистую
    const div = document.createElement('div');
    div.className = 'dynamic-modal-overlay';
    // ...
    return div;
}
```

**Применено в:**
- ✅ `cafeteria.js` - категории и назначения
- ✅ `users.js` - создание и редактирование пользователей
- ✅ `groups.js` - управление группами

### Исправления API роутинга

**Проблема:** GET /api/users/:id не работал
**Решение:** Добавлена проверка `pathParts[1]` в `api.php`

```php
if ($method === 'GET') {
    if (isset($pathParts[1])) {
        $controller->getUserById($pathParts[1]); // Один пользователь
    } else {
        $controller->getAllUsers(); // Все пользователи
    }
}
```

### Исправления часовых поясов

**Проблема:** UTC время vs локальное время (Казахстан UTC+5)
**Решение:** Расчет offset в `loadTodayTransactions()`

```javascript
const now = new Date();
const offset = now.getTimezoneOffset() * 60000;
const localISOTime = (new Date(now - offset)).toISOString().slice(0, 10);
```

### Исправления URL параметров

**Проблема:** Двойной `?` в URL (api.php?path=cafeteria/transactions?from=...)
**Решение:** Использование `&` вместо `?`

```javascript
// БЫЛО: cafeteria/transactions?from=...
// СТАЛО: cafeteria/transactions&from=...
```

### Cache-busting для JavaScript

**Проблема:** Браузер держит старые версии JS файлов
**Решение:** Добавлены версионные параметры

```html
<script src="js/cafeteria.js?v=2"></script>
<script src="js/users.js?v=2"></script>
<script src="js/groups.js?v=2"></script>
<script src="js/student-card.js?v=2"></script>
```
